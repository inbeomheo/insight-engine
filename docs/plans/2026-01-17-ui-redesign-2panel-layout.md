# UI 리디자인: 2-Panel 레이아웃

## 개요

Insight Engine의 UI/UX를 개선하여 사용성을 높이고 콘텐츠 영역을 확보한다.

### 해결할 문제
- 리포트 카드가 너무 길고 간격이 넓음
- 사이드바가 공간을 많이 차지함
- 네비게이션 구조가 불명확함
- 입력과 결과가 분리되어 스크롤 필요

### 해결 방안
- 2-Panel 레이아웃 (입력 | 결과)
- 사이드바를 아이콘 바로 축소
- 리포트 카드를 아코디언 방식으로 변경

---

## 전체 레이아웃 구조

```
┌──────┬─────────────────────┬───────────────────────────┐
│ 아이콘│     입력 패널        │       결과 패널            │
│  바  │      (40%)          │        (60%)              │
│      │                     │                           │
│ 🏠   │  URL 입력           │  리포트 카드 목록          │
│ 📜   │  AI 모델 선택        │  (아코디언 형태)           │
│ 📊   │  스타일 선택         │                           │
│ ⚙️   │  고급 옵션           │                           │
│      │  [생성 버튼]         │                           │
│      │                     │                           │
│ 👤   │                     │                           │
└──────┴─────────────────────┴───────────────────────────┘
  48px        ~40%                   ~60%
```

### 아이콘 바 (48px 고정)
- Dashboard, History, Usage, Settings 아이콘
- 하단에 로그인/프로필
- 호버 시 툴팁으로 라벨 표시
- 클릭 시 해당 기능 모달 또는 패널 내 전환

### 입력 패널 (40%)
- 현재 입력 폼들을 세로로 컴팩트하게 배치
- 스크롤 없이 한 화면에 모든 옵션 표시
- 하단에 "콘텐츠 생성" 버튼 고정

### 결과 패널 (60%)
- 생성된 리포트들이 아코디언으로 쌓임
- 독립적으로 스크롤 가능

## 리포트 카드 디자인

### 접힌 상태 (기본)
```
┌─────────────────────────────────────────────────────┐
│ ▶ 블로그 스타일로 분석한 결과                    📋 🗑️ │
│   "AI 기술의 미래: 2025년 전망과 핵심 트렌드"          │
│   AI 기술이 급속도로 발전하면서 우리 일상에...        │
│   ───────────────────────────────────────────────   │
│   Blog • Gemini 2.5 Flash • 3분 전 • 1,245자        │
└─────────────────────────────────────────────────────┘
```
- 높이: ~80px
- 제목 + 미리보기 2줄 + 메타정보

### 펼친 상태 (클릭 시)
```
┌─────────────────────────────────────────────────────┐
│ ▼ 블로그 스타일로 분석한 결과                    📋 🗑️ │
│   "AI 기술의 미래: 2025년 전망과 핵심 트렌드"          │
├─────────────────────────────────────────────────────┤
│                                                     │
│   (전체 콘텐츠 표시)                                 │
│   ...                                               │
│                                                     │
├─────────────────────────────────────────────────────┤
│   [HTML 복사] [마크다운 복사] [재생성]               │
│   Blog • Gemini 2.5 Flash • 3분 전 • 1,245자        │
└─────────────────────────────────────────────────────┘
```

### 스타일 명세
- 카드 간 간격: 8px
- 액션 버튼: 접힌 상태에서도 복사/삭제 가능
- 메타정보: 스타일, 모델, 시간, 글자수를 한 줄로

## 아이콘 바

```
┌────┐
│ 🏠 │ ← Dashboard (현재 화면)
│ 📜 │ ← History (히스토리 모달)
│ 📊 │ ← Usage (사용량 모달)
│ ⚙️ │ ← Settings (설정 모달)
│    │
│    │
│ 👤 │ ← 로그인/프로필
└────┘
```

### 스타일 명세
- 너비: 48px 고정
- 배경: #16191d (메인 배경과 동일)
- 아이콘 색상: 비활성 #9ca3af, 활성/호버 #0f758a
- 호버 시 오른쪽에 라벨 툴팁 표시

---

## 입력 패널

```
┌─────────────────────┐
│ 🔗 URL 입력         │ ← 상단 고정
│ ┌─────────────────┐ │
│ │ URL 입력창      │ │
│ │ [+추가] [분석]  │ │
│ └─────────────────┘ │
├─────────────────────┤
│ 🤖 AI 모델          │ ← 드롭다운 2개
│ 서비스 | 모델       │
├─────────────────────┤
│ 🎨 스타일           │ ← 버튼 그룹
│ [Blog][Summary]...  │
├─────────────────────┤
│ ⚙️ 고급옵션 (접힘)   │ ← 아코디언
├─────────────────────┤
│ [✨ 콘텐츠 생성]     │ ← 하단 고정
└─────────────────────┘
```

### 구성 요소
- 각 섹션 사이 구분선으로 시각적 분리
- 고급옵션은 기본 접힘 상태
- 생성 버튼은 항상 보이도록 하단 고정

---

## 인터랙션 & 동작

### 콘텐츠 생성 흐름
1. 왼쪽 패널에서 URL 입력 + 옵션 설정
2. "콘텐츠 생성" 클릭
3. 오른쪽 패널 상단에 로딩 카드 표시
4. 완료 시 자동으로 펼쳐진 상태로 결과 표시
5. 새 결과 생성 시 기존 카드는 자동으로 접힘

### 카드 인터랙션
- 카드 아무 곳 클릭 → 펼침/접힘 토글
- 📋 클릭 → 클립보드에 복사 (접힌 상태에서도 가능)
- 🗑️ 클릭 → 삭제 확인 후 제거

### 애니메이션
- 카드 펼침/접힘: 200ms ease-out
- 새 카드 등장: 위에서 슬라이드 (300ms)

---

## 반응형 처리

### 모바일 (768px 이하)
- 2-Panel → 탭 전환 방식
- "입력" 탭 / "결과" 탭으로 전환
- 아이콘 바 → 하단 네비게이션 (현재와 동일)

---

## 구현 순서

1. **Phase 1**: 레이아웃 구조 변경
   - 사이드바를 아이콘 바로 축소
   - 2-Panel 구조 적용

2. **Phase 2**: 입력 패널 재구성
   - 컴팩트한 입력 폼 배치
   - 하단 고정 생성 버튼

3. **Phase 3**: 리포트 카드 개선
   - 아코디언 방식 구현
   - 접힌/펼친 상태 디자인

4. **Phase 4**: 반응형 처리
   - 모바일 탭 전환 구현
   - 하단 네비게이션 유지
